# Agent 提示词模板 (prompt_templates.md)

> 财务报表分析 Agent 的 LLM 提示词模板库
> 版本：v1.0
> 用途：Agent 在生成报告文本、解释指标、识别风险时调用

---

## 一、系统级提示词

### 1.1 Agent 角色定义

```
你是一名专业的财务报表分析 Agent，基于张新民、钱爱民《财务报表分析（第6版）》
的"企业财务状况质量分析理论"构建。

你的核心能力：
1. 从战略视角（资产结构看经营战略，资本结构看融资战略）分析企业财务状况
2. 运用质量分析四维度（资产质量、资本结构质量、利润质量、现金流量质量）穿透财务数据表象
3. 系统计算和分析偿债能力、营运能力、盈利能力、成长能力四大类指标
4. 运用杜邦分析体系分解净资产收益率的驱动因素
5. 基于风险规则库自动识别财务异常信号

你的工作原则：
- 所有定量分析基于确定性计算工具完成，你仅负责解读和叙述
- 每个判断必须引用具体指标作为依据
- 数据缺失时标注"数据缺失"，不编造数据
- 使用"提示XX风险"而非"肯定存在XX问题"的审慎表述
- 报告中的结论标注置信度（高/中/低）
- 不确定时明确说明不确定
```

### 1.2 分析框架速查

```
分析框架：
战略分析 → 会计质量判断 → 四表质量分析 → 专项能力分析 → 综合分析与评价 → 风险提示

核心维度 - 张新民财务状况质量分析四维度：
1. 资产质量：资产真实吗？能变现吗？有效率吗？
2. 资本结构质量：钱从哪里来？融资结构健康吗？
3. 利润质量：利润真实吗？可持续吗？主业强吗？
4. 现金流量质量：现金从哪来？能自给自足吗？
```

## 二、指标计算提示词

### 2.1 指标计算结果解读

```
你收到以下指标计算结果（JSON格式）：

{ratio_results}

请完成以下任务：
1. 识别每个指标的方向（正向/反向/中性）
2. 判断每个指标当前值处于什么水平（优秀/良好/正常/偏低/危险）
3. 标记异常指标（与正常范围显著偏离的）
4. 对于异常指标，结合趋势和行业对比给出初步解释

注意事项：
- 不同行业同一指标的合理区间不同，必须结合行业均值判断
- 单期数据信息有限，必须结合趋势判断
- 如果行业数据未提供，则仅基于历史趋势和通用标准判断
- 输出格式：每个指标一行，格式为"指标名称：当前值 → 判断 → 简要解释"
```

### 2.2 指标公式与含义解释

```
用户询问指标"{ratio_name}"的含义。

请用简洁的语言解释：
1. 这个指标衡量什么（一句话）
2. 计算公式是什么
3. 数值高/低分别说明什么
4. 这个指标有什么局限性

格式要求：
- 总分结构，先给结论再展开
- 使用具体数字示例帮助理解
- 不超过200字
```

## 三、分析段落生成提示词

### 3.1 资产质量分析段落

```
你正在分析{company_name}的资产质量。请根据以下数据生成一段专业的分析文字。

资产结构数据：{asset_structure_data}
资产质量指标：{asset_quality_indicators}
趋势数据：{trend_data}
行业对比：{peer_data}

要求：
1. 从张新民"经营性资产vs投资性资产"的战略分类入手，判断企业是经营主导型还是投资主导型
2. 分析货币资金质量（是否存在大额受限资金、利息收入是否合理）
3. 分析应收账款质量（账龄分布、周转变化、坏账准备充分性）
4. 分析存货质量（周转变化、跌价准备、结构合理性）
5. 分析非流动资产质量（固定资产成新率、在建工程合理性、商誉风险）
6. 给出资产质量综合评价（优秀/良好/一般/关注/风险）

风格约束：
- 专业但可读，避免过度学术化
- 每个判断引用具体数据
- 使用"可能""提示"等适度审慎的措辞
- 禁止使用"不是...而是..."句式
- 篇幅控制在300-500字
```

### 3.2 资本结构分析段落

```
你正在分析{company_name}的资本结构质量。请根据以下数据生成一段专业的分析文字。

负债结构数据：{liability_structure_data}
所有者权益结构：{equity_structure_data}
偿债能力指标：{solvency_indicators}
融资成本数据：{financing_cost_data}
行业对比：{peer_data}

要求：
1. 运用张新民"输血vs造血"框架分析所有者权益内部结构
2. 分析经营性负债vs金融性负债的构成，判断企业融资策略
3. 分析负债期限结构，评估短期偿债压力和再融资风险
4. 判断企业是否存在"短债长投"的期限错配
5. 给出资本结构质量综合评价

风格约束：
- 专业但可读
- 每个判断引用具体数据
- 篇幅控制在300-500字
```

### 3.3 利润质量分析段落

```
你正在分析{company_name}的利润质量。请根据以下数据生成一段专业的分析文字。

利润表数据：{income_statement_data}
利润结构分解：{profit_breakdown}
非经常性损益明细：{nonrecurring_details}
毛利率与核心利润率趋势：{margin_trends}
利润含金量数据：{profit_quality_data}
行业对比：{peer_data}

要求：
1. 先分析收入质量（收入现金比率、增长质量）
2. 再分析利润结构（核心利润vs非经常性损益、毛利率变化）
3. 分析费用的合理性（各项费用率变化）
4. 用利润含金量验证利润的真实性
5. 给出利润质量综合评价

风格约束：
- 专业但可读
- 每个判断引用具体数据
- 篇幅控制在300-500字
```

### 3.4 杜邦分析解释

```
你收到了{company_name}的杜邦分析结果：

ROE = {roe}% = 净利率({npm}%) × 总资产周转率({tat}次) × 权益乘数({em}倍)

历史趋势数据：{dupond_trends}
行业对比数据：{peer_dupont}

请完成：
1. 使用连环替代法，量化每个因素对ROE变动的贡献
2. 判断ROE的驱动模式（利润驱动型/周转驱动型/杠杆驱动型）
3. 评估这种驱动模式的可持续性和风险
4. 与行业对比，判断公司在行业中的盈利模式定位

格式：
- 先给出连环替代法的数值结果
- 再给出驱动模式判断
- 最后给出可持续性评估
- 篇幅控制在250-400字
```

### 3.5 综合分析总结

```
你正在为{company_name}撰写财务报表分析的综合评价。请基于以下全部分析结果生成总结：

各维度评估：
- 资产质量评级：{aq_grade} —— {aq_summary}
- 资本结构评级：{cs_grade} —— {cs_summary}
- 利润质量评级：{pq_grade} —— {pq_summary}
- 现金流质量评级：{cf_grade} —— {cf_summary}
- 偿债能力评级：{sol_grade} —— {sol_summary}
- 营运能力评级：{op_grade} —— {op_summary}
- 盈利能力评级：{pr_grade} —— {pr_summary}
- 成长能力评级：{gr_grade} —— {gr_summary}

风险信号：{risk_signals}

要求：
1. 给出总体财务健康度评级（优秀/良好/一般/关注/风险）
2. 概括2-3项核心优势
3. 指出2-3个主要问题或风险
4. 给出1-2条建设性建议
5. 综合判断企业是否值得投资（站在投资者视角）、是否值得授信（站在债权人视角）

风格约束：
- 总结性语言，避免重复前文细节
- 结论明确但不武断
- 篇幅控制在250-400字
```

## 四、风险识别提示词

### 4.1 风险规则触发后的分析提示词

```
以下风险规则被触发，请生成专业、审慎的风险提示文本：

触发规则：{rule_id} - {rule_name}
涉及指标：{indicators}
触发条件：{condition_description}
实际数据：{actual_data}
历史趋势：{trend_data}
行业数据：{peer_data}（如有）

要求：
1. 用1-2句话解释该风险背后的会计逻辑（为什么这个条件预示风险）
2. 说明当前数据的严重程度（轻微偏离/中等偏离/严重偏离）
3. 列出建议读者进一步关注的具体事项（至少2条可操作的检查项）
4. 风险提示应采用"提示XX风险"的措辞而非"肯定存在XX问题"
5. 如果行业数据可用，判断是否存在行业共性（全行业都这样则风险降低）

风格约束：
- 准确、审慎、可操作
- 不制造不必要的恐慌
- 篇幅控制在150-250字
```

### 4.2 审计意见异常提示词

```
该公司的审计报告出具了非标准审计意见：

审计意见类型：{opinion_type}
关键审计事项：{kam}
强调事项段：{emphasis_paragraph}
审计机构：{auditor}
审计费用：{audit_fee}

请：
1. 解释该审计意见类型的含义和严重程度
2. 解读关键审计事项/强调事项段对财务分析的具体影响
3. 评估该意见对后续分析可靠性的影响
4. 给出分析建议（是否建议继续基于此财务报表进行分析）

风格约束：
- 严肃、专业、不回避问题
- 篇幅控制在200-300字
```

## 五、异常情况处理提示词

### 5.1 数据缺失处理

```
以下分析环节因数据缺失无法完成：

缺失项：{missing_items}
影响的分析环节：{affected_sections}

请生成一则简洁的数据缺失说明，包含：
1. 列出了哪些数据缺失（简要）
2. 说明了哪些分析环节因此受限
3. 给出了获取缺失数据的建议（如：请联系数据库管理员补充XX数据）
4. 告知读者当前的替代方案（例如：将使用有限数据完成，但结论置信度降低）

篇幅控制在100-150字
```

### 5.2 极端值/异常值处理

```
在计算指标时发现以下极端值：

指标名称：{ratio_name}
异常时期：{period}
数值：{value}
正常范围参考：{normal_range}
可能原因：{possible_reasons}

请生成一则该指标异常值的说明，包含：
1. 标注该值显著偏离正常范围
2. 分析可能的原因（会计准则变更/一次性事件/并购/数据错误等）
3. 建议用户关注的方向
4. 如果是因为分母接近零导致的，特别说明

篇幅控制在100-200字
```

### 5.3 勾稽关系不平

```
数据校验发现以下勾稽关系存在差异：

报表：{statement}
差异项目：{item}
差异金额：{amount}元
差异比例：{pct}%
可能原因：
  - 会计准则变更（近年准则变化较大时常见）
  - 合并范围变动
  - 数据提取或科目映射错误
  - 重分类调整

请生成简短说明，告知用户该差异的存在及其对分析的影响评估。
篇幅控制在100-150字
```

## 六、报告格式化提示词

### 6.1 Markdown 表格生成

```
请将以下数据分析结果转换为Markdown格式的表格：

数据：{data_json}
表格要求：
- 列对齐
- 数值保留1-2位小数
- 百分比数值显示%符号
- 同比/环比变化用↑↓箭头标注方向
- 异常值用**加粗**标记
- 缺失值用"—"表示
```

### 6.2 趋势描述文本

```
根据以下多期数据，请用一段简洁文字描述变化趋势：

指标名称：{ratio_name}
各期数值：{values}
各期同比变化：{changes}

要求：
- 先判断趋势方向（持续上升/持续下降/先升后降/先降后升/震荡）
- 再给出关键时间节点（拐点出现在哪一时期）
- 最后用1句话总结整体变化（"整体呈现XX趋势，期间最大变化幅度为XX个百分点"）
- 如果方向不一致（有的时期升有的降），先描述总体再分段
- 篇幅控制在100-150字
```

## 七、免责与限制说明提示词

### 7.1 通用免责声明

```
请生成分析报告的免责声明段落。
要求：
1. 说明本报告由AI Agent自动生成
2. 声明分析结果基于公开数据和教材框架，仅供研究参考
3. 不构成任何投资建议
4. 提示分析框架本身的局限性
5. 建议读者独立做出投资决策
篇幅控制在80-120字
```

### 7.2 数据局限性说明

```
以下分析存在数据局限性，请生成相关说明：

数据局限：{data_limitations}
分析受限部分：{constrained_sections}
建议的补充分析：{supplementary}

要求：
- 诚实说明数据限制对分析结论可靠性的影响
- 不建议读者忽视这些限制
- 给出如果数据完整时应该补充的分析
篇幅控制在100-200字
```
