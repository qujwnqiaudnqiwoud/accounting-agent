# Agent 工作流与工具调用说明 (agent_workflow.md)

> 财务报表分析 Agent 主控工作逻辑
> 版本：v1.1
>
> 说明：本文件区分“本课程项目已实现路径”和“后续扩展路径”。当前系统以可复现、可校验的财务报表分析为核心，模型负责规划与报告组织，财务数字和规则判断由确定性 Python 工具完成。

---

## 一、Agent 架构

```
┌─────────────────────────────────────────────────────┐
│                    主控 Agent (Orchestrator)          │
│  - 意图识别与任务规划                                  │
│  - 工具调用与结果汇总                                  │
│  - 异常处理与降级策略                                  │
│  - 最终报告生成                                       │
└──────────┬──────────────────────────────────────────┘
           │
     ┌─────┴──────────────────────────────┐
     │          工具层 (Tools Layer)        │
     │  ┌──────────┐ ┌──────────┐         │
     │  │数据读取   │ │数据清洗   │  ...    │
     │  └──────────┘ └──────────┘         │
     └─────────────────────────────────────┘
```

## 二、已实现数据类型判断

Agent 首先识别用户输入的类型：

```
输入类型                      → 处理路径
─────────────────────────────────────────────────────
PDF文件（年报）               → financial_statement_extractor 解析为标准CSV/Excel
Word文件（年报）              → financial_statement_extractor 解析为标准CSV/Excel
结构化数据 (CSV/Excel)        → 直接解析、标准化并校验
PDF + 结构化数据              → 结构化数据为主，PDF文本作为报告背景增强
```

股票代码自动拉取、同行横向比较和行业数据库接入属于后续扩展能力，当前版本不把这些功能作为已实现能力承诺。

## 三、工具调用顺序

### 3.1 当前已实现路径

```
Step 1: llm_model                   → 读取 OpenAI-compatible API 配置
Step 2: main_agent                  → smolagents ToolCallingAgent 规划工具链；失败时降级为 LLM 计划或固定管线
Step 3: file_loader                 → 读取已确认的标准 CSV/Excel
Step 4: pdf_parser                  → 可选解析 PDF 文本背景
Step 5: data_cleaner                → 科目标准化映射
Step 6: data_validator              → 核心科目完整性、资产负债表勾稽、利润现金流匹配性校验
Step 7: ratio_calculator            → 计算偿债、营运、盈利、成长、现金流量质量指标
Step 8: trend_analyzer              → 多期趋势分析
Step 9: dupont_analyzer             → 杜邦三因素分解
Step 10: cashflow_analyzer          → 现金流量质量专项判断
Step 11: risk_detector              → 内置风险规则扫描
Step 12: chart_generator            → 生成趋势图和风险分布图
Step 13: report_generator           → 生成 Markdown/Word 财务分析报告
```

### 3.2 后续扩展路径

以下模块在知识库中保留为课程项目的升级路线，不作为当前版本已实现功能：

- `audit_opinion_reader`：提取审计意见、关键审计事项和强调事项。
- `annual_report_summarizer`：提取 MD&A 中的业务、战略和主要风险。
- `structure_analyzer`：资产结构、资本结构和战略匹配分析。
- `asset_quality_analyzer`：应收、存货、商誉、长期资产等资产质量专项分析。
- `earnings_quality_analyzer`：收入、费用、减值和非经常性损益质量分析。
- `peer_comparator`：接入行业/同行数据后的横向比较。
- `consolidation_analyzer`：合并与母公司报表差量分析。
- `comprehensive_evaluator`：结合以上模块形成更完整的综合评级。

### 3.3 简化路径

当用户仅需要特定分析时：

- "分析格力电器的偿债能力" → Step 1-2 + Step 4（仅偿债指标）+ Step 5-6 + Step 9
- "格力电器的杜邦分析" → Step 1-2 + Step 4(dupont_analyzer) + Step 9
- "对格力电器做风险扫描" → Step 1-2 + Step 4 + Step 7 + Step 9

## 四、数据缺失处理策略

| 缺失情况 | 处理方式 | 用户通知 |
|---|---|---|
| 缺少某个报表项目 | 标记"数据缺失"，跳过依赖该科目的指标 | 报告中标注：*数据缺失，无法计算* |
| 期间数据不足（<3期） | 趋势分析降级为简单对比 | 报告中标注：*仅有X期数据，趋势分析受限* |
| 行业数据不可获取 | 跳过横向比较步骤 | 报告中标注：*行业数据暂无法获取，本节分析跳过* |
| 报表附注信息不全 | 定性分析标注限制 | 报告中标注：*附注信息不足，XX分析无法开展* |
| 无法匹配的科目名称 | 放入未映射列表 | 提示用户确认或手动映射 |
| 勾稽关系不平 | 标记差异金额及可能原因 | 提示用户确认是否为准则变更或数据问题 |
| 分母为零导致指标无法计算 | 返回"N/A" | 报告中标注：*分母为0，无法计算* |

## 五、避免大模型幻觉的策略

| 策略 | 实施方式 |
|---|---|
| **定量计算与LLM分离** | 所有指标计算由Python确定性函数完成，LLM仅负责解读和叙述 |
| **强制标注数据来源** | 每个数据点回溯到"哪个报表/哪个科目/第几期" |
| **不确定时明确标注** | 使用"基于XX指标的综合判断"替代绝对化表述 |
| **不编造数据** | 数据缺失时标注"数据缺失"而非自行填补 |
| **引用出处** | 行业标准引用标注来源（教材/数据库/行业报告） |
| **风险提示而非定罪** | 使用"提示XX风险"而非"肯定存在XX问题" |
| **置信度标注** | 对推定结论标注置信度（高/中/低置信度） |

## 七、异常处理机制

### 7.1 分级异常处理

| 异常级别 | 示例 | 处理方式 |
|---|---|---|
| INFO | 某个次要科目缺失 | 静默处理，在数据质量报告中记录 |
| WARNING | 缺少1年数据、某个行业数据不可用 | 降级处理，在报告对应章节标注 |
| ERROR | 缺少核心利润表数据、勾稽不平>5% | 暂停分析，提示用户补充或确认 |
| CRITICAL | 非标审计意见 | 标注风险，仍继续但所有结论附前提 |

### 7.2 异常恢复策略

- **重试机制**：数据获取失败时自动重试1次，间隔5秒
- **降级策略**：完整路径不可用时，自动降级到简化路径
- **部分结果交付**：即使某些步骤失败，已完成步骤的结果仍输出

## 八、报告生成逻辑

### 8.1 模板填空机制

`report_generator` 维护一个带占位符的Markdown模板（见 `report_template.md`），每个占位符映射到具体的数据字段：
- `{{公司名称}}`、`{{股票代码}}` 等 → 公司基本信息
- `{{ta_1}}`、`{{roe_3}}` 等 → 指标计算结果
- `{{asset_quality_summary}}` 等 → LLM生成的叙述文本
- `{{#each high_risks}}` 等 → 风险扫描循环输出

### 8.2 叙述文本生成

对于需要自然语言表述的段落，LLM基于结构化分析结果生成叙述：
- 输入：该段的指标数据 + 趋势判断 + 行业对比结果
- 输出：2-5句专业、准确的分析文字
- 约束：禁止编造数据、必须引用指标来源、不确定时使用"可能""提示"等措辞

### 8.3 数据来源标注规则

- 每个指标值后标注：`（数据来源：XX年资产负债表/利润表/现金流量表，原始科目：XX；计算公式：XX）`
- 趋势比较时标注：`（比较期间：XX年度至XX年度，共N期）`
- 行业比较时标注：`（行业数据来源：XX；可比公司：A、B、C，共N家）`
- 风险提示标注：`（触发规则：[规则编号] [规则名称]，触发条件：[具体条件描述]）`
